# Monetise modernisation diary — 2026-07-01

## Context

Production still runs in Docker (`frostickle/thalia:1.0.4`) with the old Thalia multiplex layout (`/usr/app/Thalia/websites/monetise`). Local dev has been moved to **Thalia 1.1.x from GitHub** via **bun**, as a **standalone** site — same pattern as homelab and nexus.

## Breaking changes in Thalia 1.1

1. **Controller signature** — was `function (controller)`; now `(res, req, website, requestInfo)`.
2. **Database** — Thalia loads Drizzle via `config.database.schemas` + `drizzle.config.ts`. No `db_bootstrap` ingestion.
3. **Views/partials** — loaded from `src/` (not top-level `views/`).
4. **Removed helpers** — `setHandlebarsContent`, `readAllViews`, `routeFile`, `Thalia.Controller`.

### Database connection

| Setting | Local dev | Docker (`NODE_ENV=docker`) |
|---------|-----------|----------------------------|
| Host | `localhost` | `db` |
| Port | `5233` | `3306` |
| Database | `monetise` | `monetise` |
| User | `monetise` | `monetise` |
| Password | `monetise_password` | `monetise_password` |
| URL | `mysql://monetise:monetise_password@localhost:5233/monetise` | same with `db:3306` |

### Operator checklist

```bash
docker compose up db -d    # start MariaDB
bun db:migrate             # apply drizzle/0000_init.sql
bun dev                    # or bun start
bun typecheck && bun test  # verify
```

Site runs without DB (visitor tracking disabled; `/visitors` returns "Database unavailable"). With MariaDB up and migrated, `/visitors` and `/websites` use real data.
