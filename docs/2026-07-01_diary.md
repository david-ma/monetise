## Breaking changes in Thalia 1.1

1. **Controller signature** — was `function (controller)` with `controller.db`, `controller.response`, `controller.routeFile`, etc. Now `(res, req, website, requestInfo)`.
2. **No `db_bootstrap` ingestion** — Thalia only loads Drizzle via `config.database` + `drizzle.config.ts`. Monetise must **own its Sequelize init** and import it in controllers.
3. **`SeqObject` is Drizzle-only** — monetise should use a local `MonetiseDb` type, not Thalia's `SeqObject`.
4. **Views/partials** — loaded from `src/` (not top-level `views/`). Templates move to `src/*.hbs`.
5. **Removed helpers** — `setHandlebarsContent`, `readAllViews`, `routeFile`, `Thalia.Controller` no longer exist.

## Smoke test (before migration)

- `bun thalia` starts in standalone mode (~127ms).
- `/` returns 200 only when static `public/index.html` is served; **controllers return 500** because they still expect the old API.
- No `config.database` → Thalia skips DB init; Sequelize in `db_bootstrap.ts` was never wired to controllers.
